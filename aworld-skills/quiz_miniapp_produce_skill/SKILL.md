# 问答测试类灵光小程序制作指南 (Quiz Mini-App Production Skill)

本指南为 AI 智能体构建“问答测试/人格测试”类小程序的**标准工作流**。请严格按以下阶段顺序执行。

## 第一阶段：需求分析与测试设计 (Design Phase)

### 1. 前置动作：流行趋势调研 (Mandatory)
- 在设计前，**必须**调用 `whole_search` 工具，检索以下 4 个方向的内容，每个方向至少检索 5 条相关内容，确保覆盖面和深度，为后续的测试设计提供丰富的素材和灵感：

  1. **流行梗/热梗**：
   - 当前社交媒体（小红书、抖音、微博）上流行的网络用语、表情包、梗
   - 示例：「社恐」、「E人/I人」、「emo了」、「整顿职场」、「emotional damage」等
   - 搜索关键词：「2026年流行梗」、「网络热梗」、「年轻人流行语」

  2. **流行画风/设计风格**：
   - 当前测试类应用常用的视觉风格（Low-Poly 3D、扁平插画、像素风、复古漫画等）
   - UI/UX 设计趋势（莫兰迪色系、赛博朋克、Y2K复古风等）
   - 搜索关键词：「2026设计趋势」、「测试类H5设计」、「小程序视觉风格」

  3. **优秀设计案例**：
   - 高分享率的测试类应用案例（MBTI测试、职业人格测试、古代身份测试等）
   - 爆款结果图的设计特点（排版、配色、文案风格）
   - 搜索关键词：「爆款测试H5」、「高分享率测试设计」、「测试结果图案例」

  4. **目标用户喜好**：
   - Z世代（95后-10后）的审美偏好、幽默风格、表达方式
   - 搜索关键词：「Z世代审美」、「年轻人喜欢的设计」

- 将以上调研结果总结至 `trend_research.txt`。
- **强制应用**：在后续的测试标题、金句、标签设计、视觉风格选择和文案风格设计中，必须融入调研到的流行梗（如“社恐”、“整顿职场”、“emo”等），拒绝爹味和严肃说教。

### 2. 核心设计规范
输出 `quiz_design.json`，必须包含以下结构与业务逻辑。

#### 2.1 应用基础信息
```text
【应用名称】：[填写应用名称，如"职业人格测试"]
【应用副标题】：[填写副标题，如"发现最适合你的职业方向"]
【主题风格】：[选择主题风格：ancient（古风）/modern（现代）/tech（科技）/cute（可爱）/business（商务）]
```

**命名策略（爆款要点）⭐**：
- **借势营销（概念包装）**：巧妙借用超级流行符号的认知惯性
  - 示例：「WBTI」（World Cup MBTI）借用 MBTI 的认知
  - 示例：「古代职业 MBTI」结合古风和心理测试
- **去门槛化（降低参与门槛）**：
  - 明确告知"不需要专业知识"、"不是考试"
  - 示例：「不猜冠军，也不装懂战术，只问你看球到底怎么疯」

#### 2.2 属性维度设计
定义 3-5 个属性维度，每个维度都是用户的某个特质或能力：

```text
【属性维度】：
1. [属性1名称]（如"逻辑思维"）
   - 描述：[属性含义说明]
   - 取值范围：0-30
   
2. [属性2名称]（如"社交能力"）
   - 描述：[属性含义说明]
   - 取值范围：0-30
   
3. [属性3名称]（如"创造力"）
   - 描述：[属性含义说明]
   - 取值范围：0-30
   
[根据需要添加更多属性，建议3-5个]
```

**属性设计要点：**
- 属性之间应该相互独立，不要有明显的正相关或负相关
- 属性名称要简洁（2-4 个字），易于理解
- 属性应该能够区分不同的结果类型

#### 2.3 结果类型设计
定义 6-10 种结果类型，每种结果对应特定的属性空间：

```text
【结果类型】：

结果1：[结果名称]（如"软件工程师"）
- 结果ID：result_1
- 标题：[一句话描述，如"理性思维的代码诗人"]
- 副标题：[补充性描述，增加层次感，如"用逻辑编织数字世界"]
- 属性范围：
  - [属性1名称]：20-30（高）
  - [属性2名称]：0-15（中低）
  - [属性3名称]：10-20（中）
- 核心特征：[描述这类人的核心特点，3-5 句话，要求：]
  - 第一句：总体概括，点明核心人格特质
  - 第二句：具体场景描述，让用户有画面感
  - 第三句：情感共鸣，触动用户内心
  - 第四句（可选）：未来展望或行动建议
- 金句：[**强烈推荐项**，专属金句或座右铭，要求：]
  - 类型1：诗意哲理型 - 如"代码是我的画笔，逻辑是我的诗歌"
  - 类型2：行动宣言型 - 如"用一行代码，改变一个世界"
  - 类型3：自我定义型 - 如"我不是机器，我是让机器理解人类的人"
  - 类型4：幽默反讽型 - 如"BUG 不可怕,可怕的是没有 BUG 的人生"
- 专属标签：[3-5个短标签，如"逻辑狂人", "效率至上", "完美主义", "独行侠"]
- 适合的"你"：[用"你"开头的 2-3 句话，增强代入感]
  - 如："你享受在深夜里与代码对话的时光，键盘的敲击声是你的交响乐"

结果2：[结果名称]（如"设计师"）
- 结果ID：result_2
- 标题：[一句话描述]
- 副标题：[补充性描述]
- 属性范围：
  - [属性1名称]：8-18（中低）
  - [属性2名称]：15-25（中高）
  - [属性3名称]：20-30（高）
- 核心特征：[按上述结构编写]
- 金句：[强烈推荐项，选择合适的类型]
- 专属标签：[3-5个]
- 适合的"你"：[2-3句话]

[继续添加更多结果类型...]
```

**结果类型设计要点：**
- 每个结果的属性范围应该有明显差异，形成不同的"属性画像"
- 所有结果的属性范围组合应该覆盖整个属性空间（避免盲区）
- **核心特征必须生动具体，采用"总-分-情感-展望"结构**
- **每个结果最好有独特且令人印象深刻的金句**
- **专属标签要简洁有力，能快速传达人格特质**
- **"适合的你"部分要用第二人称，增强用户的代入感和认同感**

**标签化与自嘲艺术（爆款要点）⭐**：
- **"无公害的自我调侃"**：标签应采用轻微喜剧色彩，既能彰显个性，又不会引发社交反感
  - ✅ 好的标签：「熬夜硬撑王」「假摔影帝」「战术嘴替」「摸鱼仙人」
  - ❌ 避免：过于严肃或负面的标签（如「社交废物」「智商堪忧」）
- **活人感文案**：避免生硬的"AI味"，文案要像"朋友精准补刀"
  - 示例：「凌晨三点不是夜，是你的开球时间」
  - 示例：「你的拖延症晚期，但总能在deadline前一秒拯救世界」

#### 2.4 题目设计
设计 8-15 道题目，每道题目包含问题和 2-4 个选项：

```text
【题目列表】：

题目1：
问题：[填写问题文本，简洁明了]
选项A：[选项文本]
  - [属性1名称]：+3
  - [属性2名称]：+0
  - [属性3名称]：+1
选项B：[选项文本]
  - [属性1名称]：+0
  - [属性2名称]：+3
  - [属性3名称]：+2
选项C：[选项文本]（可选）
  - [属性1名称]：+1
  - [属性2名称]：+1
  - [属性3名称]：+3

题目2：
问题：[填写问题文本]
选项A：[选项文本]
  - [属性1名称]：+2
  - [属性2名称]：+1
  - [属性3名称]：+0
选项B：[选项文本]
  - [属性1名称]：+0
  - [属性2名称]：+2
  - [属性3名称]：+3

[继续添加更多题目...]
```

**题目设计要点：**
- **题目要像活人聊天（用户体验核心）⭐**：
  - 问题要有画面感和情绪张力，不要像问卷调查
  - ✅ 好的题目：「你脑内弹幕是？」「主队输了你会沉默还是开麦？」「爆冷了你第一反应是什么？」
  - ❌ 避免：「你的性格特点是什么？」「你喜欢团队合作吗？」
- 问题应该简洁、直接，避免歧义
- 每个选项的属性加分应该有明显的倾向性（体现不同的人格特质）
- 加分值建议在 0-5 之间，总和建议控制在 3-8 之间
- 每道题目至少应该对 2 个属性产生影响
- 题目应该均匀覆盖所有属性维度，避免某个属性被过度或不足地测试
- **多维加分机制**：每个选项不应只导向单一结果，而应同时给 2-3 个人格加分，让结果显得更有分析深度

### 3. JSON 输出骨架 (Skeleton Template)
请严格参考此精简结构生成 `quiz_design.json`：

## 第二阶段：项目初始化与配置同步 (Project Setup)

1. **阅读前置规范**：
   - 必须仔细阅读 `lingguang-miniapp-meta-skill.md`，确保遵循灵光小程序的底层开发规范。
2. **复制模板**：使用 `bash` 将 `~/.aworld/workspaces/demo_workspace_for_quiz` 复制为 `~/.aworld/workspaces/workspace`。
2. **数据注入**：使用 `bash` 编写并执行 Python/Shell 脚本，读取 `quiz_design.json`，将数据精准注入到 `workspace/data/storyData.ts` 中。

   **⚠️ 注入前接口校验（必须执行）**：
   在编写 Python 脚本注入数据到 `storyData.ts` 之前，**必须**先执行以下命令读取接口定义：
   `cat workspace/data/storyData.ts | grep -A 15 "export interface Question"`
   `cat workspace/data/storyData.ts | grep -A 15 "export interface ResultItem"`
   仔细阅读模板中的 `⚠️ AI生成注意` 注释，确保你编写的注入脚本输出的字符串与 TypeScript 接口完全一致。

   - 替换 `APP_CONFIG` 占位符。
   - 利用 `// @INJECT_QUESTIONS_START` 和 `// @INJECT_RESULTS_START` 锚点精准替换 `QUESTIONS` 和 `RESULTS` 数组。
   
   **应用基础信息APP_CONFIG部分**：
   - appName: `{{APP_NAME}}` → 应用名称appName
   - appSubtitle: `{{APP_SUBTITLE}}` → 应用副标题appSubtitle
   - theme: `{{THEME}}` → 主题风格theme（ancient/modern/tech/cute/business）
   - coverImage: `coverImg` → 封面图路径（稍后生成）
   - backgroundImage: `bgImg` → 背景图路径（稍后生成）
   - attributeNames: `[{{ATTRIBUTE_NAMES}}]` → attributes列表中每个元素的name，如 `["逻辑思维", "社交能力", "创造力", "执行力"]`
   - attributeMax: `{{ATTRIBUTE_MAX}}` → 属性最大值，通常为 30
   
   **配置权重**：
   - `MATCH_WEIGHTS` 替换：模板中默认为 `export const MATCH_WEIGHTS: number[] = [1.0, 1.0, 1.0];`。你必须在 Python 注入脚本中，根据 `quiz_design.json` 中 `attributes` 的长度，动态生成对应长度的权重数组（如 `[1.0, 1.0, 1.0, 1.0]`），并使用正则表达式替换掉模板中的默认值。

   **题目数组部分 (`QUESTIONS`)**：
   将 `{{QUESTIONS_ARRAY}}` 替换为完整的题目数组。

   **结果数组部分 (`RESULTS`)**：
   将 `{{RESULTS_ARRAY}}` 替换为完整的结果数组。

3. **一致性校验**：确保 `storyData.ts` 中的属性名称在 Config、Questions、Results 中**完全一致**（含大小写）。

## 第三阶段：视觉资产生成 (Image Generation)

本阶段负责生成测试应用所需的所有图片资源，包括封面、背景和结果图。

### 1. 调用工具与并行策略
- 使用 `image_generator` 子智能体生成所有图片。
- **支持并行**：最多可并行调用 **5个** `image_generator` 任务以提高效率。你可以直接并行调用，无需使用async_spawn_subagent__spawn_parallel这个工具，这个工具不稳定。切记，最多5个。

### 2. 图片规格与画风要求
#### 2.1 基础规格
- **尺寸**：严格遵循 3:4 宽高比，分辨率为 `768x1024`。
- **格式**：PNG 格式，支持透明背景（结果图建议使用）。
- **文件命名**：
  - 封面图：`cover.png`
  - 背景图：`background.png`
  - 结果图：`result_1.png`, `result_2.png`, ... （按结果 ID 命名）

#### 2.2 画风统一要求
- **画风必须高度统一**：所有图片必须使用相同的艺术风格（如：扁平插画、渐变风、赛博朋克、古风水墨等）。
- **主题契合**：画风应与测试主题契合（如古风测试使用水墨风，科技测试使用赛博朋克风）。
- **色调一致**：整体色调应该协调统一，建议选择 2-3 种主色调贯穿所有图片。

#### 2.3 各类图片的具体要求

**封面图 (`cover.png`)**：
- 吸引眼球、突出主题
- **必须包含应用主题的字样**：在生成封面图的 Prompt 中，必须明确要求 AI 将应用的主题/标题以艺术字的形式融入画面中（例如："with bold typography saying '你的古代身份测试'"）。
- 构图居中，画面饱满，突出核心视觉元素，文字需清晰可见且与画风融合。
- 示例：职业测试的封面可以是多个职业人物的剪影，上方带有“职业性格测试”的字样。

**背景图 (`background.png`)**：
- 低饱和度、低对比度，作为整个应用的背景
- 不要有过于复杂的细节（避免干扰前景内容）
- 可以是渐变、纹理、抽象图案
- 建议透明度在 30-50%，确保前景文字可读

### 3. 提示词 (Prompt) 构造公式
   为了确保出图质量，请严格按照以下公式拼接 `content`：

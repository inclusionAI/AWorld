# 问答测试类灵光小程序制作指南 (Quiz Mini-App Production Skill)

本指南旨在为 AI 智能体提供一套标准化的工作流，用于从零开始构建"问答测试"、"人格测试"或"属性评估"类的灵光小程序。请严格按照以下流程和规范执行。

## 第一阶段：需求分析与测试设计 (Design Phase)

本阶段的核心目标是指导策划与 AI 在设计测试时，如何规范、细致地输出测试设计文档（`quiz_design.txt`）。核心是确保输出的设计文档具有**清晰的属性维度**、**合理的题目分布**，并完美契合"多维度属性匹配"的核心机制。

### 0. 流行趋势调研（必需）⭐

**重要前提**：在开始设计测试之前，**必须先进行流行趋势调研**，确保应用内容有趣、幽默、贴近当下年轻人的喜好。

#### 0.1 调研目标

通过 **deep search（深度搜索）** 检索以下内容：

1. **流行梗/热梗**：
   - 当前社交媒体（小红书、抖音、微博）上流行的网络用语、表情包、梗
   - 示例：「社恐」、「E人/I人」、「emo了」、「整顿职场」、「emotional damage」等
   - 搜索关键词：「2026年流行梗」、「网络热梗」、「年轻人流行语」

2. **流行画风/设计风格**：
   - 当前测试类应用常用的视觉风格（Low-Poly 3D、扁平插画、像素风、复古漫画等）
   - UI/UX 设计趋势（莫兰迪色系、赛博朋克、Y2K复古风等）
   - 搜索关键词：「2026设计趋势」、「测试类H5设计」、「小程序视觉风格」

3. **优秀设计案例**：
   - 高分享率的测试类应用案例（MBTI测试、职业人格测试、古代身份测试等）
   - 爆款结果图的设计特点（排版、配色、文案风格）
   - 搜索关键词：「爆款测试H5」、「高分享率测试设计」、「测试结果图案例」

4. **目标用户喜好**：
   - Z世代（95后-10后）的审美偏好、幽默风格、表达方式
   - 搜索关键词：「Z世代审美」、「年轻人喜欢的设计」

#### 0.2 调研方法

**使用 deep search 工具**：
```
调用 deep_search 或 web_search 工具，针对以上 4 个方向分别检索：
- 「2026年最新网络流行梗」
- 「测试类应用设计趋势 2026」
- 「高分享率人格测试案例分析」
- 「Z世代喜欢的幽默风格」
```

**输出调研报告**：
将检索到的内容整理成 `trend_research.txt`，包含：
- 3-5个当前最流行的梗及其使用场景
- 2-3种当前流行的视觉风格及特点
- 2-3个优秀案例的设计亮点分析
- 目标用户的喜好总结（幽默风格、表达方式、价值观）

#### 0.3 将调研结果融入设计

**在后续设计中应用调研成果**：
- **金句创作**：融入流行梗和年轻化表达（如「社恐不是病，是能量管理的艺术」）
- **标签设计**：使用网感十足的标签（如「摸鱼仙人」、「emo体质」、「整顿大师」）
- **视觉风格选择**：选择调研到的流行画风（如 Low-Poly 3D、Y2K 复古风）
- **文案风格**：采用年轻人喜欢的幽默方式（自嘲、反差萌、认知反转）

**示例应用**：
- 调研发现「emotional damage」流行 → 金句中使用「你不是容易受伤的小朋友，你是 emotional damage 收集者」
- 调研发现 Low-Poly 3D 风格火爆 → 第三阶段选择 Low-Poly 作为主视觉风格
- 调研发现「整顿职场」是热梗 → 结果类型中加入「职场整顿官」、标签用「#整顿大师」

**执行要求**：
✅ **在输出测试设计文档之前，必须先完成流行趋势调研** ⭐  
✅ 将调研结果保存到 `trend_research.txt`  
✅ 在设计中明确标注哪些内容受调研启发（如：金句使用了「emo」梗，来源于调研）

### 1. 核心设计理念

- **多维度属性系统**：测试必须基于 N 个属性维度（如：逻辑思维、社交能力、创造力等），每个维度有明确的取值范围（通常 0-30）。
- **题目驱动评分**：每道题目的每个选项都会对不同属性产生加分/减分效果，引导用户最终落在特定的属性空间。
- **确定性匹配**：基于用户的最终属性得分，通过距离算法匹配最接近的结果类型，确保相同答案得到相同结果。
- **结果覆盖度**：所有结果类型的属性范围必须覆盖整个属性空间（0-max），避免用户得分无法匹配任何结果。

### 2. 测试设计规范

在输出 `quiz_design.txt` 时，**必须且只能**包含以下核心设计内容，严格按照以下结构编写：

#### 2.1 应用基础信息
```text
【应用名称】：[填写应用名称，如"职业人格测试"]
【应用副标题】：[填写副标题，如"发现最适合你的职业方向"]
【主题风格】：[选择主题风格：ancient（古风）/modern（现代）/tech（科技）/cute（可爱）/business（商务）]
【题目数量】：[填写题目总数，建议 8-15 题]
【结果类型数量】：[填写结果类型数量，建议 6-10 种]
```

**命名策略（爆款要点）⭐**：
- **借势营销（概念包装）**：巧妙借用超级流行符号的认知惯性
  - 示例：「WBTI」（World Cup MBTI）借用 MBTI 的认知
  - 示例：「古代职业 MBTI」结合古风和心理测试
  - **为什么有效**：用户一看就懂，极大降低教育成本
- **去门槛化（降低参与门槛）**：
  - 明确告知"不需要专业知识"、"不是考试"
  - 示例：「不猜冠军，也不装懂战术，只问你看球到底怎么疯」
  - **为什么有效**：让伪球迷、凑热闹的泛大众也能无压力参与

#### 2.2 属性维度设计
定义 3-5 个属性维度，每个维度都是用户的某个特质或能力：

```text
【属性维度】：
1. [属性1名称]（如"逻辑思维"）
   - 描述：[属性含义说明]
   - 取值范围：0-30
   
2. [属性2名称]（如"社交能力"）
   - 描述：[属性含义说明]
   - 取值范围：0-30
   
3. [属性3名称]（如"创造力"）
   - 描述：[属性含义说明]
   - 取值范围：0-30
   
[根据需要添加更多属性，建议3-5个]
```

**属性设计要点：**
- 属性之间应该相互独立，不要有明显的正相关或负相关
- 属性名称要简洁（2-4 个字），易于理解
- 属性应该能够区分不同的结果类型

#### 2.3 结果类型设计
定义 6-10 种结果类型，每种结果对应特定的属性空间：

```text
【结果类型】：

结果1：[结果名称]（如"软件工程师"）
- 结果ID：result_1
- 标题：[一句话描述，如"理性思维的代码诗人"]
- 副标题：[补充性描述，增加层次感，如"用逻辑编织数字世界"]
- 属性范围：
  - [属性1名称]：20-30（高）
  - [属性2名称]：0-15（中低）
  - [属性3名称]：10-20（中）
- 核心特征：[描述这类人的核心特点，3-5 句话，要求：]
  - 第一句：总体概括，点明核心人格特质
  - 第二句：具体场景描述，让用户有画面感
  - 第三句：情感共鸣，触动用户内心
  - 第四句（可选）：未来展望或行动建议
- 金句：[**必需项**，专属金句或座右铭，要求：]
  - 类型1：诗意哲理型 - 如"代码是我的画笔，逻辑是我的诗歌"
  - 类型2：行动宣言型 - 如"用一行代码，改变一个世界"
  - 类型3：自我定义型 - 如"我不是机器，我是让机器理解人类的人"
  - 类型4：幽默反讽型 - 如"BUG 不可怕,可怕的是没有 BUG 的人生"
- 专属标签：[3-5个短标签，如"#逻辑狂人 #效率至上 #完美主义 #独行侠"]
- 适合的"你"：[用"你"开头的 2-3 句话，增强代入感]
  - 如："你享受在深夜里与代码对话的时光，键盘的敲击声是你的交响乐"

结果2：[结果名称]（如"设计师"）
- 结果ID：result_2
- 标题：[一句话描述]
- 副标题：[补充性描述]
- 属性范围：
  - [属性1名称]：8-18（中低）
  - [属性2名称]：15-25（中高）
  - [属性3名称]：20-30（高）
- 核心特征：[按上述结构编写]
- 金句：[必需项，选择合适的类型]
- 专属标签：[3-5个]
- 适合的"你"：[2-3句话]

[继续添加更多结果类型...]
```

**结果类型设计要点：**
- 每个结果的属性范围应该有明显差异，形成不同的"属性画像"
- 所有结果的属性范围组合应该覆盖整个属性空间（避免盲区）
- 结果数量建议 6-10 种，太少缺乏趣味性，太多难以区分
- **核心特征必须生动具体，采用"总-分-情感-展望"结构**
- **金句是必需项，不是可选项，每个结果必须有独特且令人印象深刻的金句**
- **专属标签要简洁有力，能快速传达人格特质**
- **"适合的你"部分要用第二人称，增强用户的代入感和认同感**

**标签化与自嘲艺术（爆款要点）⭐**：
- **"无公害的自我调侃"**：标签应采用轻微喜剧色彩，既能彰显个性，又不会引发社交反感
  - ✅ 好的标签：「熬夜硬撑王」「假摔影帝」「战术嘴替」「摸鱼仙人」
  - ❌ 避免：过于严肃或负面的标签（如「社交废物」「智商堪忧」）
- **活人感文案**：避免生硬的"AI味"，文案要像"朋友精准补刀"
  - 示例：「凌晨三点不是夜，是你的开球时间」
  - 示例：「你的拖延症晚期，但总能在deadline前一秒拯救世界」

#### 2.4 题目设计
设计 8-15 道题目，每道题目包含问题和 2-4 个选项：

```text
【题目列表】：

题目1：
问题：[填写问题文本，简洁明了]
选项A：[选项文本]
  - [属性1名称]：+3
  - [属性2名称]：+0
  - [属性3名称]：+1
选项B：[选项文本]
  - [属性1名称]：+0
  - [属性2名称]：+3
  - [属性3名称]：+2
选项C：[选项文本]（可选）
  - [属性1名称]：+1
  - [属性2名称]：+1
  - [属性3名称]：+3

题目2：
问题：[填写问题文本]
选项A：[选项文本]
  - [属性1名称]：+2
  - [属性2名称]：+1
  - [属性3名称]：+0
选项B：[选项文本]
  - [属性1名称]：+0
  - [属性2名称]：+2
  - [属性3名称]：+3

[继续添加更多题目...]
```

**题目设计要点：**
- **题目要像活人聊天（用户体验核心）⭐**：
  - 问题要有画面感和情绪张力，不要像问卷调查
  - ✅ 好的题目：「你脑内弹幕是？」「主队输了你会沉默还是开麦？」「爆冷了你第一反应是什么？」
  - ❌ 避免：「你的性格特点是什么？」「你喜欢团队合作吗？」
- 问题应该简洁、直接，避免歧义
- 每个选项的属性加分应该有明显的倾向性（体现不同的人格特质）
- 加分值建议在 0-5 之间，总和建议控制在 3-8 之间
- 每道题目至少应该对 2 个属性产生影响
- 题目应该均匀覆盖所有属性维度，避免某个属性被过度或不足地测试
- **多维加分机制**：每个选项不应只导向单一结果，而应同时给 2-3 个人格加分，让结果显得更有分析深度

### 4. 执行要求 (Action Required)

**重要流程**：
1. **第一步**：使用 deep search 完成流行趋势调研（参见 0. 流行趋势调研）
2. **第二步**：基于调研结果，按照【标准化输出模板】生成测试设计内容
3. **第三步**：将设计内容写入 `quiz_design.txt`

**无需**在文件中包含复杂的 UI 布局描述或技术实现细节，**只保留最核心的、与测试设计相关的内容**。

### 5. 设计验证清单

在完成设计后，请自我检查以下内容：

✅ **已完成流行趋势调研，并将结果保存到 `trend_research.txt`（必需）** ⭐  
✅ **金句、标签、文案融入了调研到的流行梗和年轻化表达** ⭐  
✅ 属性维度是否清晰且相互独立？  
✅ 结果类型的属性范围是否覆盖整个属性空间？

## 第三阶段：视觉资产生成 (Image Generation)

本阶段负责生成测试应用所需的所有图片资源，包括封面、背景和结果图。

### 1. 调用工具与并行策略
- 使用 `image_generator` 子智能体生成所有图片。
- **支持并行**：最多可并行调用 **5个** `image_generator` 任务以提高效率。直接并行调用即可，无需使用 async_spawn_subagent__spawn_parallel 工具。切记，最多5个。

### 2. 图片规格与画风要求

#### 2.1 基础规格
- **尺寸**：严格遵循 3:4 宽高比，分辨率为 `1024x1536`。
- **格式**：PNG 格式，支持透明背景（结果图建议使用）。
- **文件命名**：
  - 封面图：`cover.png`
  - 背景图：`background.png`
  - 结果图：`result_1.png`, `result_2.png`, ... （按结果 ID 命名）

#### 2.2 画风统一要求
- **画风必须高度统一**：所有图片必须使用相同的艺术风格（如：扁平插画、渐变风、赛博朋克、古风水墨等）。
- **主题契合**：画风应与测试主题契合，例如：
  - 职业测试 → 现代商务风格
  - 古代人物测试 → 中国古风水墨画
  - 科技测试 → 赛博朋克或科技感插画
  - 性格测试 → 温暖柔和的插画风格
- **色调一致**：整体色调应该协调统一，建议选择 2-3 种主色调贯穿所有图片。

#### 2.3 各类图片的具体要求

**封面图 (`cover.png`)**：
- 吸引眼球、突出主题
- 可以包含应用名称的视觉元素（但不要直接包含文字）
- 构图居中或上部留白（方便叠加标题）
- 示例：职业测试的封面可以是多个职业人物的剪影

**背景图 (`background.png`)**：
- 低饱和度、低对比度，作为整个应用的背景
- 不要有过于复杂的细节（避免干扰前景内容）
- 可以是渐变、纹理、抽象图案
- 建议透明度在 30-50%，确保前景文字可读

## 常见问题 (FAQ)

### Q1: 如何确保结果类型的属性范围覆盖了整个属性空间？

**A**: 使用"网格法"验证覆盖度：
1. 将属性空间分成若干个网格（如每个属性分成 3 段：低 0-10、中 10-20、高 20-30）
2. 列举所有可能的属性组合（如：低低低、低低中、低低高...）
3. 对每个组合，检查是否至少有一个结果类型的属性范围包含它

**示例**：
- 属性：A、B、C，范围 0-30
- 结果1：A[20-30], B[0-10], C[0-10] → 覆盖"高低低"
- 结果2：A[0-10], B[20-30], C[0-10] → 覆盖"低高低"
- 结果3：A[0-10], B[0-10], C[20-30] → 覆盖"低低高"
- ...确保所有 27 种组合（3^3）都被覆盖

### Q2: 题目数量和结果类型数量如何确定？

**A**: 
- **题目数量**：建议 8-15 题
  - 太少（< 5 题）：区分度不够，结果不准确
  - 太多（> 20 题）：用户疲劳，完成率低
  - 最佳：10-12 题，既有足够的区分度，又不会太长
- **结果类型数量**：建议 6-10 种
  - 太少（< 4 种）：缺乏趣味性，用户容易猜到结果
  - 太多（> 12 种）：难以区分，属性空间覆盖困难
  - 最佳：8 种，足够丰富又容易设计

### Q3: 属性加分值如何设计？

**A**:
- 每个选项的总加分建议在 **3-8** 之间
- 单个属性的加分建议在 **0-5** 之间
- 每道题目应该至少对 **2 个**属性产生影响
- 避免所有选项加分完全一样（失去区分度）

**示例**：
```typescript
// 好的设计
{
  text: "选项A",
  scores: { "逻辑思维": 3, "社交能力": 0, "创造力": 1 } // 总和 4，倾向逻辑
}
{
  text: "选项B",
  scores: { "逻辑思维": 0, "社交能力": 3, "创造力": 2 } // 总和 5，倾向社交+创造
}

// 不好的设计
{
  text: "选项A",
  scores: { "逻辑思维": 1, "社交能力": 1, "创造力": 1 } // 总和 3，无倾向
}
{
  text: "选项B",
  scores: { "逻辑思维": 1, "社交能力": 1, "创造力": 1 } // 总和 3，无倾向，与选项A无区别
}
```

### Q4: 如何测试匹配算法的正确性？

**A**:
1. **边界测试**：全选第一个选项，全选最后一个选项，看是否匹配到预期结果
2. **确定性测试**：相同选择应该得到相同结果，测试 2-3 次
3. **覆盖度测试**：随机选择多次，确保所有结果类型都能被匹配到
4. **距离验证**：手动计算一次匹配距离，与代码结果对比

### Q5: 图片生成失败或画风不统一怎么办？

**A**:
1. **提示词优化**：确保所有图片的提示词都包含相同的"共同风格描述"
2. **分批生成**：不要一次性生成所有图片，分批生成可以更容易保持一致性
3. **参考图**：如果有条件，提供一张参考图给 image_generator，要求"保持与参考图相同的画风"
4. **重新生成**：如果某张图片画风偏差太大，单独重新生成

1. **第一阶段**：**先进行流行趋势调研（必需）⭐**，然后设计清晰的属性维度、题目和结果类型，确保覆盖度和一致性
2. **内容创作强化**：✨ **金句创作、文案写作、标签设计** - 融入流行梗和年轻化表达，提升结果页吸引力（第一阶段补充）
3. **第二阶段**：复制模板代码，填充三个配置文件，验证一致性
4. **第三阶段**：生成统一画风的图片资源（封面、背景、结果图），选择调研到的流行风格，强调美观度和趣味性
5. **第四阶段**：**生成背景音乐并实现开关控制（必需）** ⭐，音效可选
6. **第五阶段**：代码集成、匹配算法测试、样式微调、**分享功能实现（必需）** ⭐
7. **第六阶段**：遵循全局约束，特别是灵光小程序元规范

**成功关键**：
- ✅ **完成流行趋势调研，内容紧跟潮流（必需）** ⭐
- ✅ 属性维度设计合理，相互独立
- ✅ 结果类型覆盖整个属性空间
- ✅ 题目设计均衡，区分度明显
- ✅ **金句有记忆点，融入流行梗，能让用户产生分享欲望** ⭐
- ✅ **核心特征有共鸣，让用户感到"这说的就是我"** ⭐
- ✅ **结果图美观且有趣，使用流行画风，包含细节彩蛋和氛围感** ⭐
- ✅ **分享功能完善，降低分享摩擦，提升传播率** ⭐
- ✅ 图片画风统一，主题契合
- ✅ 代码符合灵光小程序元规范
- ✅ 测试充分，匹配算法确定性

**差异化竞争力（爆款要素）**：
- 💎 **借势营销** - 借用超级流行符号（如 MBTI）的认知惯性，降低教育成本
- 💎 **内容有梗有趣** - 通过 deep search 掌握最新流行趋势，让应用永远年轻化
- 💎 **金句不是可选，是必需** - 每个结果都有独特且令人难忘的金句，作为社交货币
- 💎 **标签化与自嘲** - "无公害的自我调侃"，既能彰显个性，又不会引发社交反感
- 💎 **文案有温度** - 使用"四句式"结构和"你"开头，营造对话感和共鸣，避免"AI味"
- 💎 **卡中卡设计** - 采用"白底外框 + 彩色内框"嵌套结构，激发收集欲和炫耀心理
- 💎 **道具叙事** - 用视觉符号传达人格精髓（熬夜拿咖啡、假摔拿面具），用户一眼就懂
- 💎 **三层字体层级** - 超大主标签 + 装饰性副标题 + 诊断金句，确保朋友圈缩略图可读
- 💎 **视觉有细节** - 结果图不只是人物，而是有故事感的场景，包含彩蛋元素
- 💎 **SSR稀有款** - 低概率稀有结果激发炫耀心理，引爆二次传播
- 💎 **题目像聊天** - 有画面感和情绪张力，不像问卷调查
- 💎 **体验要快** - 进度提示、大按钮、单手操作，极致优化移动端体验
